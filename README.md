# Cyclistic Bike Share Case Study

## Overview
This project analyzes Cyclistic bike-share data to understand differences between casual riders and annual members.

## Tools Used
- Google Sheets
- SQL (BigQuery)
- Python (pandas)
- Tableau (optional visualization)

## Process
- Cleaned ride data by filtering invalid trips (<1 min and >1440 min)
- Created ride duration and day of week features
- Analyzed rider behavior using aggregation techniques
- Compared member vs casual usage patterns

## Key Insights
- Members take more consistent rides throughout the week
- Casual riders tend to take longer rides
- Casual usage peaks on weekends

## Recommendations
- Target casual riders with weekend membership promotions
- Encourage conversion of frequent casual users to annual plans
- Design pricing strategies around long-duration riders
